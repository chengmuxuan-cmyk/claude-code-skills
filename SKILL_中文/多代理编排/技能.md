# 多代理编排

## 概览

将多代理系统视为受约束的生产系统，而不是松散助手集合。在任何代理开始工作前，先定义角色契约、上下文形态、权限、工具、MCP 访问、记录与清理。

## 源码锚点

- `src/tools/AgentTool/`
- `src/tools/AgentTool/runAgent.ts`
- `src/tools/AgentTool/built-in/`

## 工作流

1. 决定哪些工作必须保留在主关键路径，哪些可以委派。
2. 为每种代理类型写明确契约：目标、允许工具、默认模型、异步或同步行为、权限策略。
3. 为每个代理选择上下文策略：完整分叉、瘦身分叉或仅任务提示。
4. 限定权限作用域，避免父会话授权意外泄露给子代理。
5. 执行前预加载正确技能、frontmatter 钩子与代理专属 MCP 服务器。
6. 记录转录、元数据、父子层级与进度消息，确保过程可审计、可恢复。
7. 定义停止条件，如 `maxTurns`、中止传播与完成回调。
8. 在 `finally` 中清理 MCP 客户端、会话钩子、提示词缓存跟踪、文件状态与孤儿待办项。

## 设计规则

- 最终责任在主代理，委派不等于责任转移。
- 区分同步协作与异步后台工作，因为权限行为不同。
- 代理专属 MCP 视为增量连接，仅清理该代理创建的连接。
- 对只读代理移除无关继承上下文，如陈旧 git 状态或无用大策略文件。
- 持久化足够元数据，便于恢复路径重建正确代理形态。
- 将技能预加载视为启动流程的一部分，而非可选优化。

## 失败模式

- 把关键路径阻塞工作委派出去，主代理反而空转。
- 子代理继承了原本不该给它的 allow 规则。
- 异步代理走了必须交互确认的权限路径，无法完成。
- 未记录旁路转录，恢复连续性丢失。
- 跳过清理，遗留活跃 MCP 连接或泄漏状态。

## 输出

- 产出代理矩阵，包含角色、权限、工具、模型与退出条件。
- 产出委派策略，说明何时本地处理、何时委派、何时并行。
- 产出生命周期检查清单，覆盖创建、进度、恢复、中止与清理。
