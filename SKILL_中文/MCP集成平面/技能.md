# MCP 集成平面

## 概览

将 MCP 视为外部能力总线，而不是轻量 RPC 包装。它必须管理连接类型、认证、发现、超时、进度、超大输出、会话过期与用户引导式询问，同时不扰乱主代理循环。

## 源码锚点

- `src/services/mcp/client.ts`
- `src/services/mcp/config.ts`
- `src/services/mcp/elicitationHandler.ts`

## 工作流

1. 先标准化服务器配置，并区分 stdio、SSE、HTTP、WebSocket、IDE 与代理型传输。
2. 为每类传输构建正确的认证提供器、请求头、代理行为与超时策略。
3. 将远程认证失败转换为明确 `needs-auth` 状态，而非笼统连接错误。
4. 记忆化连接，并以有界 LRU 缓存工具、资源与命令发现结果。
5. 发现阶段清理 Unicode、裁剪超长描述，并保留只读/破坏性等工具注解。
6. 执行 MCP 工具调用时，启用进度上报、硬超时、`_meta` 保留与大结果持久化。
7. 对 URL 引导式询问，让钩子或用户先完成响应，再重试工具调用。
8. 对 401、404 与会话过期路径执行定向恢复，如清缓存、重连或刷新令牌。
9. 清理阶段关闭传输、停止进程监控并注销全部清理处理器。

## 设计规则

- 每次请求创建新的超时控制，不复用陈旧中止信号。
- 对长生命周期 SSE 订阅流，不应用普通请求超时。
- 日志仅记录脱敏且去查询参数后的基础 URL。
- 对超大输出使用持久化引用或截断指引，而不是直接膨胀提示词。
- 区分认证失败、传输关闭与会话过期，因为恢复路径不同。
- 将 MCP 工具元数据视为权限、UI、上下文压缩与搜索行为的共享输入。

## 失败模式

- 一个过期令牌触发一连串 401，并被缓存成长期不可用。
- 复用连接级超时，导致后续请求全部瞬时失败。
- 把会话过期 404 当作普通网络故障处理。
- 工具描述无限增长，挤爆上下文窗口。
- 大型 MCP 响应未持久化或截断就直接注入转录。

## 输出

- 产出覆盖认证、超时与清理行为的传输矩阵。
- 产出 needs-auth、会话过期、超时与断连的恢复策略。
- 产出输出策略，定义何时截断、何时持久化、何时保留 `_meta`。
