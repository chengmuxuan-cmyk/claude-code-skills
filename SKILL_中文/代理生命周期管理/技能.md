# 代理生命周期管理

## 概览

把代理生命周期设计成完整的运行时协议：创建、选择模型、塑造上下文、组装工具、启动钩子、流式汇报进度、记录转录并进行确定性清理。缺少任何一个阶段，子代理都会从“效率倍增器”变成“泄漏来源”。

## 源码锚点

- `src/tools/AgentTool/runAgent.ts`

## 工作流

1. 分配稳定的代理 ID，并解析模型、工作目录与转录子目录。
2. 谨慎分叉上下文，复用前先移除父级未完成的工具调用。
3. 按代理类型裁剪继承上下文，避免只读代理携带大量无关状态。
4. 构建代理专属应用状态，覆盖权限模式、允许工具、提示行为与推理强度。
5. 加载启动资源，如 `SubagentStart` 钩子、frontmatter 钩子、预加载技能与代理专属 MCP 服务器。
6. 创建子代理上下文，并决定哪些回调与中止控制器共享，哪些隔离。
7. 在执行全过程记录转录消息、元数据、进度与 API 指标，确保可恢复与可观测。
8. 在 `finally` 中销毁 MCP 客户端、钩子、提示词缓存跟踪、读文件缓存、转录映射与残留待办。

## 设计规则

- 区分编排与生命周期：前者决定“派谁做事”，后者决定“被派出的代理如何存活与退出”。
- 默认给异步代理独立中止控制器；同步代理共享控制，除非有明确理由。
- 为嵌套异步代理保留根状态写入路径，使其可安全更新共享状态。
- 对后续要审阅转录的代理，保留工具调用结果。
- 持久化足够的恢复元数据，确保可重建正确代理形态。
- 所有清理统一放在 `finally`，不要依赖正常完成路径。

## 失败模式

- 复用带有未完成工具调用的父消息，导致立即触发 API 错误。
- 异步代理仍走依赖交互弹窗的权限策略。
- 跳过元数据写入，恢复时无法定位正确代理类型。
- 创建了代理专属 MCP 连接却从不清理。
- 在大量已结束子代理之间遗留读文件缓存与待办映射。

## 输出

- 产出覆盖初始化、运行、中止、恢复与清理的生命周期状态图。
- 产出上下文塑形策略，定义哪些可继承、哪些必须裁剪。
- 产出清理检查清单，为每个运行时资源指定责任方。
